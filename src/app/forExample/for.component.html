@for replaced ngFor for looping over data
The for directive is a replacement for the ngFor extension. It basically does the same things but it's much more performant and doesn't cause as many repaints when processing array data. One key note is that the new syntax does require a tracking attribute in the data being itterated over. This helps the core UI processor keep track of elements inside of the loop.
https://blog.angular-university.io/angular-for/
a normal for loop
@for(chick of chickens; track chick.breedID) {
{{ chick.breedName }}
}

a for loop with no data
@for(chick of coonsInYard; track chick.breedID) {
{{ chick.breedName }}
} @empty {
